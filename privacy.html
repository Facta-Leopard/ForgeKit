ForgeKit Privacy Policy

Privacy information for ForgeKit users.

This page explains how ForgeKit handles local project data, optional integrations, credentials, support reports, and website access information. The policy is written directly on this page in multiple languages.

Short version: ForgeKit is designed around local project editing. Your project files remain under your control unless you export, share, upload, or enable an optional integration that sends data to a service you configure.

English

This Privacy Policy applies to ForgeKit Editor and the ForgeKit support pages hosted from this repository.

Information ForgeKit handles

  • ForgeKit edits project files, assets, scenes, validation reports, export settings, and release-evidence records that you create or select.
  • Local editing does not require a ForgeKit account, and the static support website does not provide a form that directly collects personal information.
  • Project data remains on your device or in folders you choose unless you export it, share it, upload it, or enable an optional integration.

Optional integrations

  • If you configure an AI provider, prompts, selected project context, model requests, and responses may be sent to the provider you choose. That provider's privacy policy applies to its processing.
  • If you enable MCP or external client workflows, local tool requests, diagnostics, and project operations may be exchanged with the clients you configure.
  • If update, licensing, crash reporting, or product feedback features are enabled and configured, those services may receive the information required for that feature, such as app version, build number, OS version, diagnostic details, license status, or feedback text.

Credentials and sensitive data

  • API keys, provider credentials, license information, and similar secrets should not be posted in public GitHub issues.
  • When the app stores credentials through system-supported secure storage, you remain responsible for the provider accounts and keys you add.

Website and support reports

  • These pages are intended to be served by GitHub Pages. GitHub may process basic access logs and repository activity according to GitHub's own policies.
  • Information you voluntarily include in a public GitHub issue is visible publicly. Remove private project content, local paths, credentials, and personal information before posting.

Data control

  • You control local project files and can delete local projects, exported packages, logs, and settings from your device.
  • Data sent to third-party services is controlled by those services and their retention policies.
  • ForgeKit does not sell personal information and does not use these support pages for advertising profiles.

Contact and changes

For privacy questions, open a GitHub issue without including secrets. This policy may be updated when ForgeKit features, integrations, or distribution channels change.

ForgeKit이 처리하는 정보

  • ForgeKit은 사용자가 만들거나 선택한 프로젝트 파일, 에셋, 씬, 검증 보고서, 내보내기 설정, 릴리스 증거 기록을 편집합니다.
  • 로컬 편집에는 ForgeKit 계정이 필요하지 않으며, 이 정적 지원 웹사이트는 개인정보를 직접 수집하는 입력 양식을 제공하지 않습니다.
  • 프로젝트 데이터는 사용자가 내보내기, 공유, 업로드를 하거나 선택적 연동을 활성화하지 않는 한 사용자의 기기 또는 사용자가 선택한 폴더에 남아 있습니다.

선택적 연동

  • AI 제공자를 설정하면 프롬프트, 선택된 프로젝트 맥락, 모델 요청, 응답이 사용자가 선택한 제공자에게 전송될 수 있습니다. 해당 처리는 그 제공자의 개인정보 정책을 따릅니다.
  • MCP 또는 외부 클라이언트 흐름을 활성화하면 사용자가 설정한 클라이언트와 로컬 도구 요청, 진단 정보, 프로젝트 작업이 오갈 수 있습니다.
  • 업데이트, 라이선스, 충돌 보고, 제품 피드백 기능을 활성화하고 설정한 경우 해당 기능에 필요한 앱 버전, 빌드 번호, OS 버전, 진단 정보, 라이선스 상태, 피드백 내용 등이 관련 서비스로 전송될 수 있습니다.

자격 증명과 민감 정보

  • API 키, 제공자 자격 증명, 라이선스 정보와 같은 민감한 값은 공개 GitHub 이슈에 올리지 마세요.
  • 앱이 시스템 보안 저장소를 통해 자격 증명을 저장하더라도, 사용자가 추가한 제공자 계정과 키에 대한 책임은 사용자에게 있습니다.

웹사이트와 지원 요청

  • 이 페이지는 GitHub Pages로 제공되도록 설계되어 있습니다. GitHub는 자체 정책에 따라 기본 접속 로그와 저장소 활동을 처리할 수 있습니다.
  • 공개 GitHub 이슈에 자발적으로 포함한 정보는 공개됩니다. 게시 전에 비공개 프로젝트 내용, 로컬 경로, 자격 증명, 개인정보를 제거해 주세요.

데이터 관리

  • 사용자는 로컬 프로젝트 파일을 관리하며, 기기에서 로컬 프로젝트, 내보낸 패키지, 로그, 설정을 삭제할 수 있습니다.
  • 제3자 서비스로 전송된 데이터는 해당 서비스와 그 보존 정책의 적용을 받습니다.
  • ForgeKit은 개인정보를 판매하지 않으며, 이 지원 페이지를 광고 프로필 작성에 사용하지 않습니다.

문의와 변경

개인정보 관련 질문은 민감한 정보를 포함하지 말고 GitHub 이슈로 남겨 주세요. ForgeKit 기능, 연동, 배포 방식이 변경되면 이 정책도 갱신될 수 있습니다.

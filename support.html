ForgeKit Support

Help and contact information.

Use this page for ForgeKit Editor support, setup questions, bug reports, export issues, validation workflow questions, MCP setup, and optional AI provider configuration support.

Public support channel: GitHub Issues. Do not post API keys, tokens, credentials, paid-license details, private project files, or confidential screenshots in public issues.

English

For ForgeKit support, open an issue in the GitHub repository. Support is provided on a best-effort basis through the public issue tracker.

What to include

  • ForgeKit version or build date, macOS version, and device model if relevant.
  • The workflow area: project creation, 2D scene editing, 3D scene editing, export, validation, release evidence, MCP, or AI provider setup.
  • Expected result, actual result, and the smallest repeatable steps.
  • Screenshots or logs only after removing secrets, private paths, credentials, license data, and unpublished project content.

Before posting

  • Check whether the project opens from a local folder you control.
  • Run the relevant validation or export workflow again and note the exact failing row or message.
  • If the issue involves MCP or AI, confirm that your transport, endpoint, provider, model, and credential reference are the intended ones.

Deutsch

Für ForgeKit-Support erstelle bitte ein Issue im GitHub-Repository. Support wird nach bestem Aufwand über den öffentlichen Issue-Tracker bereitgestellt.

Was enthalten sein sollte

  • ForgeKit-Version oder Build-Datum, macOS-Version und gegebenenfalls das Gerätemodell.
  • Betroffener Bereich: Projekterstellung, 2D-Szenenbearbeitung, 3D-Szenenbearbeitung, Export, Validierung, Release-Nachweise, MCP oder AI-Provider-Konfiguration.

  • Prüfe, ob das Projekt aus einem lokalen Ordner geöffnet wird, auf den du Zugriff hast.
  • Führe den betroffenen Validierungs- oder Exportablauf erneut aus und notiere die genaue fehlgeschlagene Zeile oder Meldung.
